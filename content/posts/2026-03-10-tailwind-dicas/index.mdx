O Tailwind permite estilizar componentes sem sair do arquivo HTML/JSX. A produtividade é incomparável quando combinado com um design system bem estruturado — classes utilitárias eliminam a alternância entre arquivos CSS e componentes, mantendo o contexto visual sempre próximo do código.

  variant?: 'default' | 'outline' | 'ghost'
}

export function Button({
  variant = 'default',
  className,
  ...props
}: ButtonProps) {
  return (

  )
}

## Steps

<Steps>
  <Step title='Instale as dependências'>
    Adicione o Tailwind e o plugin PostCSS ao projeto:

    ```bash
    bun add tailwindcss @tailwindcss/postcss
    ```
  </Step>

  <Step title='Configure o PostCSS'>
    Crie ou edite o arquivo `postcss.config.mjs` na raiz do projeto:

    ```js title="postcss.config.mjs"
    export default {
      plugins: {
        '@tailwindcss/postcss': {},
      },
    }
    ```
  </Step>

  <Step title='Importe no CSS global'>
    Adicione a diretiva de importação no topo do seu arquivo CSS principal:

    ```css title="globals.css"
    @import 'tailwindcss';
    ```
  </Step>

  <Step title='Use no projeto'>
    Agora você já pode usar as classes utilitárias nos seus componentes:

    ```tsx
    export function Card() {
      return (
        <div className='rounded-lg border bg-card p-6 shadow-sm'>
          <h2 className='text-lg font-semibold'>Pronto!</h2>
        </div>
      )
    }
    ```
  </Step>
</Steps>

## Accordion

<Accordion>
  <AccordionItem title='O que é o Tailwind CSS?'>
    É um framework CSS utilitário que fornece classes de baixo nível para construir designs
    customizados diretamente no HTML, sem a necessidade de escrever CSS personalizado.
  </AccordionItem>

  <AccordionItem title='Tailwind vs CSS Modules'>
    CSS Modules isola estilos por componente usando classes geradas. Tailwind usa
    classes utilitárias compartilhadas. A principal diferença está na colocação:
    com Tailwind, o estilo vive junto do markup, eliminando a troca de contexto.
  </AccordionItem>

  <AccordionItem title='Como funciona o purge/treeshaking?'>
    O Tailwind analisa os arquivos configurados no campo `content` do
    tailwind.config e remove todas as classes não utilizadas no build de produção,
    resultando em um CSS final extremamente pequeno.
  </AccordionItem>

  <AccordionItem title='Vale a pena usar em projetos grandes?'>
    Sim. Em projetos grandes o Tailwind escala bem porque o tamanho do CSS não cresce com o
    projeto — ele permanece constante já que as classes são reutilizadas. O maior desafio é
    manter consistência nas classes, o que pode ser resolvido com componentes reutilizáveis.
  </AccordionItem>
</Accordion>
